This bibliography collects the external references that shaped Labyrinth's direction.

It keeps the sources you introduced while describing the site, plus the one reference formalized into the brief during drafting.

It leaves out placeholder URLs, assistant-only research links, and operational repo links that did not shape the project direction.

Design Direction

  • sive.rs set the tone for direct navigation, low chrome, and editorial clarity.
  • laminae.org reinforced an essay-first reading rhythm and stronger typographic hierarchy.
  • bachtogauss.com/about gave the reference for narrow reading measure, one scale driving type and spacing, and rare media-query pivots.
  • Typographics 2020 Schedule pushed the spec toward dense but highly legible information layout.
  • jacobleech.com supported giving the site visual personality while keeping motion restrained.
  • sive.rs/su set the bar for short, root-level, slashless public URLs.
  • 101 Design Rules provided principled rules for design restraint and defensible decisions.
  • justinjackson.ca/words calibrated the lower bound for minimalism and helped define how far to go.
  • musa.bet/home.htm served as a conceptual reference for featural symbol logic and internal consistency, not the site's visual style.
  • Type Terms set the model for interactive teaching pieces and how a work might move and instruct.
  • barcia.dev showed generous breathing room and whitespace as a primary design material.
  • manuelmoreale.com showed narrow column, generous spacing, and restrained typography.
  • paco.me showed a disciplined, spacious page with minimal ornament.
  • shud.in/projects de-emphasized layout and gave content more room.
  • subpixel.space/entries pushed maximum whitespace and minimal decoration.
  • thesephist.com showed academic plainness where content commands attention.
  • danilafe.com/blog/blog_microfeatures clarified the reading microfeatures the site should have.
  • Euclid, Book I, Proposition 3 showed structured content presentation, cross-referencing, and diagrams treated as co-equal with text.
  • Printing Types gave the strongest interface and navigation reference: a book made into a digital edition with deep-linkable headings and readable cross-references.
  • Notes on the Euclid project covered typeface selection for historical fidelity, SVG structural elements, and decorative initials turned into a custom font.

Animation Direction

  • UI Ball Loaders gave the animation reference. The implementation direction still moved toward CSS and inline SVG rather than a JavaScript dependency.

Spec Direction

  • Writing a good CLAUDE.md helped shape the push toward a shorter, denser, more directive AGENTS.md and planning style.

Low-JS Direction

  • Do you really need JavaScript in 2024? strengthened the case for making no build-generated JavaScript the default.
  • How this site is coded reinforced semantic HTML, browser defaults, minimal CSS, and content-first structure over branding chrome.

Typography and CSS Technique

  • More Meaningful Typography introduced modular type scales from harmonic ratios and informed the pow()-based scale approach.
  • Math in CSS: powers documented CSS pow() and related math functions, which enabled clean single-point derivation of the type scale.
  • Fluid type on web.dev reinforced the fluid type approach and flagged the WCAG 1.4.4 zoom constraint.
  • Fluid Typography introduced continuous fluid scaling via clamp() and viewport units and shaped the no-breakpoints approach.
  • em, px, pt, cm, in… anchored the proportional units system: rem, em, and ch over px.

Brief Reference Added During Drafting

  • POSSE anchored the IndieWeb compatibility goal that was later written into the project aim.
