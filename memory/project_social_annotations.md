name: Social margin annotations feature idea
description: Future feature concept for collaborative margin notes, inspired by Curius and c82.net/printing-types. Not in MVP. Implementation path uncertain.
type: project
---

The ideal reading experience for Labyrinth includes social margin annotations — notes from friends visible inline in the margin alongside the text, rather than as a separate comments section. Reference: Curius (curius.app) does this well.

**Why:** The c82.net/printing-types digital book showed how powerful it is to have a work that cross-references other works and lets you deep-link to headings. The natural next step is: what if readers could leave traces in the margins that other readers see? This would make Labyrinth feel like a shared reading experience, not just a publishing platform.

**How to apply:** MVP ships without this. When considering future features, this is a high-priority direction. Implementation is open — options include:
- A plugin or third-party service (ideally reusing an existing login/auth solution)
- A bespoke implementation
- Promoting an existing tool that does this

The key constraint: it should feel like margin notes, not a comment thread. Inline, spatial, tied to a specific passage.
