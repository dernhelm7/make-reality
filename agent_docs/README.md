# Agent Docs

This folder contains the product aim, the planning record that shaped the spec, and the spec documents that guide implementation.

The spec docs in this folder were generated from the aim below and the planning record in `plan.md`.
Use `agent_docs/examples/` for canonical source fixtures and expected outcomes.
See `bibliography.md` for the external references that shaped the project direction.

## Aim

A personal publishing site + portfolio; a medium for thinking and a way to share ideas and make it possible for people to send me gifts (monetary or otherwise).

Editorial, typographic, restrained. It should feel more like an interactive book than a blog.

The homepage purpose is to act like the cover and title page of a book. Its contents section feels like a table of contents to my works. The works (poems, projects, art pieces) I create share the same navigation and some aesthetics but may have custom layouts.

## Idiosyncratic Design Preferences

- Static output, fast and lean
- Indie-web-compatible. Ref: POSSE https://indieweb.org/POSSE
- RSS feed
- URLs should be names, not file paths: short, direct, and slashless. Ref https://sive.rs/su
