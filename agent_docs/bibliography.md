# Project Bibliography

This bibliography collects the external URLs that came up in the Labyrinth project conversations and directly shaped the direction of the work.

It focuses on the references you introduced while describing what you wanted, plus one reference that was formalized into the product brief during spec drafting.

It leaves out placeholder URLs, assistant-only research links, and operational repo links that were not used as project-direction references.

## Design Direction

- `https://sive.rs/`
  Set the tone for direct navigation, low chrome, and editorial clarity.

- `https://laminae.org/`
  Reinforced an essay-first reading rhythm and stronger typographic hierarchy.

- `https://2020.typographics.com/schedule/`
  Pushed the spec toward dense but highly legible information layout.

- `https://jacobleech.com/`
  Supported giving the site visual personality while keeping motion restrained.

- `https://sive.rs/su`
  Set the bar for short, root-level, slashless public URLs.

- `https://wearecollins.com/story/101-design-rules/`
  Provided principled rules for design restraint and defensible decisions.

- `https://justinjackson.ca/words.html`
  Calibrated the lower bound for minimalism. Helped define how far to go.

- `https://musa.bet/home.htm`
  Conceptual reference for featural symbol logic and internal consistency, not the site's visual style.

- `https://avark.agency/typeterms`
  Set the model for interactive teaching pieces — how a work might move and instruct.

- `https://barcia.dev/`
  Visual design reference; generous breathing room and whitespace as a primary design material.

- `https://manuelmoreale.com/`
  Visual design reference; narrow column, generous spacing, restrained typography.

- `https://paco.me`
  Visual design reference; disciplined and spacious with minimal ornament.

- `https://shud.in/projects`
  Visual design reference; de-emphasized layout with content given room.

- `https://subpixel.space/entries/`
  Visual design reference; maximum whitespace, minimal decoration.

- `https://thesephist.com/`
  Visual design reference; academic plainness where content commands attention.

- `https://danilafe.com/blog/blog_microfeatures/`
  Reference for reading microfeatures — the kind of details the site should have.

- `https://www.c82.net/euclid/book1/#prop3`
  Reference for structured content presentation: consistent proposition templates, internal cross-referencing, and diagram integrated as co-equal to text rather than illustration. Navigation visuals excluded.

- `https://www.c82.net/printing-types/`
  Strong interface and navigation reference. A book made into a digital edition: cross-references link out to other books you can actually read; headings are deep-linkable. The ideal extension would be social margin annotations visible inline (à la Curius) rather than as comments.

- `https://www.c82.net/blog/?id=79`
  Design notes on the Euclid project. Covers typeface selection for historical fidelity, SVG for custom structural elements (brackets, diagrams with interactive highlighting), and decorative initials converted to a custom font.

## Animation Direction

- `https://uiball.com/ldrs/`
  Animation style reference. Implementation moved to CSS and inline SVG to avoid the JS dependency.

## Spec Direction

- `https://www.humanlayer.dev/blog/writing-a-good-claude-md`
  Helped shape the push toward a shorter, denser, more directive `AGENTS.md` and planning style.

## Low-JS Direction

- `https://the-sustainable.dev/do-you-really-need-javascript-in-2024/`
  Strengthened the case for making no build-generated JavaScript the default, with HTML and CSS carrying the site.

- `https://liamnugent.me/posts/how-this-site-is-coded/`
  Reinforced semantic HTML, browser defaults, minimal CSS, and a content-first structure over branding chrome.

## Typography and CSS Technique

- `https://alistapart.com/article/more-meaningful-typography/`
  Introduced modular type scales from harmonic ratios; informed the `pow()`-based scale approach.

- `https://danielcwilson.com/posts/mathematicss-powers/`
  Documented CSS `pow()` and related math functions; enabled clean single-point derivation of the type scale.

- `https://web.dev/articles/baseline-in-action-fluid-type`
  Reinforced the fluid type approach and flagged the WCAG 1.4.4 zoom constraint.

- `https://www.smashingmagazine.com/2016/05/fluid-typography/`
  Introduced continuous fluid scaling via `clamp()` and viewport units; shaped the no-breakpoints approach.

- `https://www.w3.org/Style/LieBos3e/em.en.html`
  Anchored the proportional units system — `rem`, `em`, `ch` over `px`.

## Brief Reference Added During Drafting

- `https://indieweb.org/POSSE`
