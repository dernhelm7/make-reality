# Look

This file defines the launch visual direction: typography, spacing, color, page rhythm, and the appearance of shared elements within the limits set by `web-rules.md`.

## Direction

The launch site must feel editorial, restrained, and book-like.

It must use:

- serif-forward typography
- generous vertical spacing
- a narrow reading measure on work pages
- a quiet palette with one muted accent

The palette must read as paper, ink, and one subdued accent:

- background should be a warm light tone close to `#f6f1e7`
- body text should be a dark tone close to `#1f1a17`
- accent elements should use a muted tone close to `#6d5a45`

## Typography

Primary text and headings must use this serif stack:

`Iowan Old Style`, `Palatino Linotype`, `Book Antiqua`, `Georgia`, `serif`

Type rules must include:

- body text sized for comfortable long-form reading
- headings that rely on scale and spacing rather than decorative styling
- summary and metadata text that remain visibly secondary to the main body

## Homepage

The homepage must read as a cover/title page.

It should:

- give the site title the strongest visual weight
- place the introduction close to the title
- make the primary link into `Works` immediately visible

## Works Index

The works index must read like a table of contents.

It should:

- emphasize section headings over decoration
- keep entry links and summaries easy to scan
- preserve clear spacing between sections and entries

## Work Page

The work page must prioritize continuous reading.

It should:

- keep the text column within a narrow measure
- separate the title, summary, and metadata from the body with spacing rather than heavy ornament
- avoid visual interruptions inside the reading column

## Navigation and Footer

The shared navigation and footer must stay visually quiet on interior pages.

The homepage must give the same navigation and footer stronger emphasis through spacing, placement, or rule treatment.

Support and contact links must remain secondary to reading content.
